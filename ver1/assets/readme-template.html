MEETING AGENT

Teams 미팅 자동 캡처

v0.2.0

1빠른 시작

STEP 1
zip 풀고 meeting-agent.exe 더블클릭
STEP 2
환영창 → 시작하기 → 저장 폴더 선택
STEP 3
트레이 상주. Teams 미팅 시작 시 알림

처음 실행 시 SmartScreen 경고 → "추가 정보" → "실행" (1회만)

2요구 환경

OSWindows 10 build 19041 (2004) 이상 · Windows 11 권장
TeamsNew Teams (Classic Teams 미지원)
WebView2Windows 11 기본 탑재 / Windows 10 거의 모두 설치됨

3사용 흐름

A 시작 실행 → 환영창 → 저장 폴더 선택
B 트레이 상주 시계 옆 마스코트 + 안내 토스트
C 미팅 감지 → 녹화 30초 응답 대기
D 녹화 중 이벤트 자막·화면공유·캡처 대상 변경 알림
E 미팅 종료 자동 감지 → 10초 확인 → 마무리
⚙ 언제든 — 트레이 우클릭 상태 보기 / 수동 시작·중지 / 설정

4흐름

상황동작
미팅 감지 시녹화 / 무시 토스트 (30초)
녹화 중 자막·공유·리사이즈이벤트 알림 팝업 (트레이 설정에서 OFF 가능)
미팅 종료 감지 시"녹화 중지 + 폴더 열기" 팝업 (10초)
10초 무응답자동 finalize → 완료 팝업

5트레이 메뉴

항목동작
▶ 녹화 시작미팅 감지 상태에서 활성
■ 녹화 중지녹화 중 활성
📁 마지막 세션 폴더 열기이전 녹화 있을 때 활성
설정 → 저장 위치 변경다음 녹화부터 즉시 반영
설정 → 감지 시 자동 녹화토스트 없이 즉시 시작
설정 → Windows 시작 시 자동 실행레지스트리 등록
설정 → 이벤트 알림자막/공유/리사이즈 알림 ON·OFF
설정 → 로그 파일 보기agent.log
✕ 종료finalize 후 종료

6결과물

<저장 폴더>\YYYY-MM-DD-HHMMSS-MeetingSession\

meeting.mp4 1080p · 6 Mbps · 10 fps · H.264 + AAC (Teams 음성 + 마이크) transcript.txt [HH:MM:SS.mmm] Speaker: text · Teams 라이브 캡션 ON 필요 agent.log 이번 세션의 디버그 로그 (문제 보고용)

7문제 해결

트레이 아이콘이 안 보여요
시계 옆 ∧ 클릭. Windows 11에선 기본 숨김.
미팅 감지 팝업이 안 떠요
New Teams인지 확인. run-debug.bat으로 콘솔 모드 → 로그 확인.
transcript.txt가 비어있어요
Teams 라이브 캡션을 켜세요. ··· → "언어 및 음성" → "라이브 캡션 켜기"
강제 종료 후 MP4가 깨졌어요
ffmpeg -i meeting.mp4 -c copy meeting-recovered.mp4
로그 위치
실시간 %TEMP%\meeting-agent.log · 세션 사본 <세션>\agent.log
