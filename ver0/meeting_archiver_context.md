# Meeting Archiver 프로젝트 - 대화 컨텍스트

이 파일을 클로드 코드에서 읽고, 이어서 구현을 진행해줘.

---

## 프로젝트 목표

봇 없이 백그라운드에서 동작하는 **Teams 회의 자동 기록 서비스** 개발
- 사용자 본인이 참여한 회의를 자동으로 아카이빙하는 용도
- 실시간 처리 불필요, 회의 종료 후 배치 처리로 충분

## 핵심 기능

### 1. 회의 전사 수집 (회의 종료 후)
- Teams 내장 Transcription 기능 활용 (회의 중 켜두면 VTT 파일 생성)
- **Microsoft Graph API**로 회의 종료 후 전사본 자동 다운로드
- 엔드포인트: `GET /me/onlineMeetings/{meetingId}/transcripts/{transcriptId}/content`
- VTT 형식, **화자 정보 + 타임스탬프** 포함
- Graph API 인증: Azure AD 앱 등록 → OAuth 토큰

### 2. 발표자료 화면 변화 감지 + 캡처
- Teams 화면 공유 영역을 주기적 스크린샷 (1초 간격)
- **Perceptual Hash(pHash)** 로 이전 프레임과 비교
- 해밍 거리가 threshold 초과 시에만 저장 (슬라이드 전환 감지)
- pHash 사용 이유: 마우스 커서 등 미세 변화는 무시하고 슬라이드 전환만 잡기 위함
- 회의 시작/종료 감지: Teams 프로세스 + 윈도우 타이틀 모니터링

### 3. 병합 및 회의록 출력
- 타임스탬프 기준으로 전사 텍스트 + 슬라이드 캡처 매칭
- Markdown/HTML 회의록 출력
- AI 요약 (Claude API 등) 추가 가능

## 기술 스택

- Python 기반 백그라운드 서비스 (시스템 트레이)
- 화면 캡처: `mss` + `imagehash` + `PIL`
- 전사 수집: Microsoft Graph API
- 캡처 저장 포맷: WebP (용량 절감), 유사 이미지 중복 제거

## 아키텍처

```
┌──────────────────────────────────────────────┐
│           Meeting Archiver Service           │
│              (Python, 시스템 트레이)            │
├──────────────────────────────────────────────┤
│                                              │
│  [회의 중]                                    │
│   Screen Monitor                             │
│   ├─ Teams 창 영역 감지 (Win32 API)           │
│   ├─ 주기적 캡처 (1초 간격)                    │
│   ├─ pHash 비교 → 변화 감지                   │
│   └─ 변화 시 스크린샷 저장 (타임스탬프)          │
│                                              │
│  [회의 후]                                    │
│   Transcript Fetcher                         │
│   ├─ Graph API로 최근 회의 조회                │
│   ├─ VTT 전사본 다운로드 (화자 포함)            │
│   └─ 파싱 → 구조화된 데이터                    │
│                                              │
│  [병합]                                       │
│   타임스탬프 기준으로 전사 + 슬라이드 매칭        │
│   → Markdown/HTML 회의록 출력                  │
│                                              │
└──────────────────────────────────────────────┘
```

- 주요 일정: ...
- 액션 아이템: ...

## 합의된 구현 순서

1. **화면 변화 감지 모듈** 프로토타입부터 시작
2. Graph API 전사 수집 모듈
3. 병합 및 회의록 생성
